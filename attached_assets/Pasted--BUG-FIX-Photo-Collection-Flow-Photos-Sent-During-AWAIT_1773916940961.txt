# BUG FIX: Photo Collection Flow — Photos Sent During AWAITING_STORE_NAME Are Silently Ignored

## Bug Description

When the agent starts a new report, the bot asks "Напиши назву торгової точки" (write the store name). The agent then sends **photos instead of a text name**. The bot does NOT respond — no "📸 Фото 1/5 отримано", no error, nothing. The photos are silently dropped.

**Expected behavior:**
The bot should either:
- (Option A — recommended) Accept the photo as implicit "start collecting", use a placeholder store name like "Без назви" or auto-extract from context, then respond with "📸 Фото 1/5 отримано. Надішли ще або натисни Готово."
- (Option B) Reply with a reminder: "⚠️ Спочатку напиши назву торгової точки, потім надсилай фото."

**What actually happens:**
Complete silence. No bot response. Photos lost. See screenshot — two photos sent after "Новий звіт" prompt, zero bot replies.

## Root Cause Analysis

The ConversationHandler likely has states like:

```
AWAITING_STORE_NAME → COLLECTING_PHOTOS → ANALYZING
```

When the bot is in `AWAITING_STORE_NAME` state and receives a **photo** (not text), the handler has no matching entry for `MessageHandler(filters.PHOTO)` in that state, so `python-telegram-bot` silently ignores the update.

## Evidence from Render Logs

Looking at successful reports (#31-#34), each shows:
```
getFile → photo downloaded
sendMessage → confirmation  
POST api.anthropic.com → analysis
Stored 1 photos for report #XX
sendMessage → report
```

But the failed interaction shows NO `getFile` call at all — the photo never gets processed because the handler doesn't catch it in `AWAITING_STORE_NAME` state.

## Required Fix

### In `bot.py` — ConversationHandler states:

**Add a photo handler to the `AWAITING_STORE_NAME` state** that gracefully handles photos arriving before the store name:

```python
# Option A (recommended): Accept photo, use placeholder name, move to COLLECTING_PHOTOS

async def handle_photo_before_name(update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Handle photo sent when bot is still waiting for store name."""
    # Use placeholder store name
    context.user_data['store_name'] = "Без назви"
    context.user_data['photos'] = []
    
    # Process this photo as the first photo
    photo = update.message.photo[-1]  # highest resolution
    file = await context.bot.get_file(photo.file_id)
    photo_bytes = await file.download_as_bytearray()
    context.user_data['photos'].append(bytes(photo_bytes))
    
    count = len(context.user_data['photos'])
    await update.message.reply_text(
        f"📸 Фото {count}/5 отримано. Надішли ще або натисни Готово."
    )
    
    return COLLECTING_PHOTOS  # transition to photo collection state
```

**Register it in the ConversationHandler:**

```python
conv_handler = ConversationHandler(
    entry_points=[...],
    states={
        AWAITING_STORE_NAME: [
            MessageHandler(filters.TEXT & ~filters.COMMAND, handle_store_name),
            # NEW: handle photos arriving before store name
            MessageHandler(filters.PHOTO, handle_photo_before_name),
        ],
        COLLECTING_PHOTOS: [
            MessageHandler(filters.PHOTO, handle_photo),
            MessageHandler(
                filters.TEXT & ~filters.COMMAND, 
                handle_done_or_comment  # handles "Готово" and text comments
            ),
        ],
        # ...
    },
    fallbacks=[...],
)
```

### Additional edge cases to handle:

1. **Multiple photos sent rapidly (burst):** Telegram sometimes sends photos as separate updates in quick succession. Each should increment the counter and respond:
   - "📸 Фото 1/5 отримано. Надішли ще або натисни Готово."
   - "📸 Фото 2/5 отримано. Надішли ще або натисни Готово."

2. **Photo sent as a document (file attachment):** Some users send photos as files, not as compressed photos. Add handler for `filters.Document.IMAGE`:
   ```python
   MessageHandler(filters.Document.IMAGE, handle_photo_as_document),
   ```

3. **6th photo rejection:** If the agent sends more than 5 photos:
   ```python
   if len(context.user_data['photos']) >= 5:
       await update.message.reply_text(
           "⚠️ Максимум 5 фото. Натисни Готово для аналізу."
       )
       return COLLECTING_PHOTOS
   ```

4. **"Готово" button not working:** Looking at the screenshot, the bot shows persistent keyboard buttons ("📸 Новий звіт", "📊 Моя статистика"). Make sure the "✅ Готово" button appears as a ReplyKeyboard when in COLLECTING_PHOTOS state:

   gotovo_keyboard = ReplyKeyboardMarkup(
