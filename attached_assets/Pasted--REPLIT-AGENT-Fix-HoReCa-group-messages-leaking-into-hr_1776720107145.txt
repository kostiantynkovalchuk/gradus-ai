## STEP 1 — SEPARATE THE WEBHOOK ENDPOINTS

The fix is to give each bot its own webhook path so the handler knows
which bot it's dealing with before doing anything else.

### Option A — Path-based separation (PREFERRED)
Register two separate routes:
- `/api/telegram/webhook/maya` → handles TELEGRAM_MAYA_BOT_TOKEN only
- `/api/telegram/webhook/gradus` → handles TELEGRAM_BOT_TOKEN only

Then re-register webhooks with Telegram:
```
TELEGRAM_MAYA_BOT_TOKEN → https://gradus-ai.onrender.com/api/telegram/webhook/maya
TELEGRAM_BOT_TOKEN      → https://gradus-ai.onrender.com/api/telegram/webhook/gradus
```

### Option B — Secret token header (if path separation is too invasive)
Telegram supports `setWebhook` with `secret_token` parameter.
The secret is sent as `X-Telegram-Bot-Api-Secret-Token` header on each request.
Check which bot sent the request by reading this header.

**Use Option A** unless the audit reveals a strong reason not to.

## STEP 3 — RE-REGISTER WEBHOOKS WITH TELEGRAM

After creating the new routes, register the correct URL for each bot.

Write a one-time script (do not add to production code):
```python
import os, requests

maya_token = os.environ['TELEGRAM_MAYA_BOT_TOKEN']
gradus_token = os.environ['TELEGRAM_BOT_TOKEN']
base = 'https://gradus-ai.onrender.com'

# Register Maya HR bot
r1 = requests.post(
    f'https://api.telegram.org/bot{maya_token}/setWebhook',
    json={'url': f'{base}/api/telegram/webhook/maya'}
)
print('Maya:', r1.json())

# Register GradusMediaBot
r2 = requests.post(
    f'https://api.telegram.org/bot{gradus_token}/setWebhook',
    json={'url': f'{base}/api/telegram/webhook/gradus'}
)
print('Gradus:', r2.json())
```

Run this script and show the output. Both should return `{"ok": true}`.
