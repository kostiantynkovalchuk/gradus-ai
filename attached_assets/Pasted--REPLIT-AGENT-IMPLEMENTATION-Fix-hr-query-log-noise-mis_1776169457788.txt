### CHANGE 2 ‚Äî Content guard before logging
**File:** `backend/routes/telegram_webhook.py`

Find the call site of `handle_hr_question` (around line 581) and the
`/api/hr/log-query` POST call (around lines 1888‚Äì1899).

Add a helper function (place it near the top of the file with other helpers):

```python
import re

def _should_log_hr_query(text: str) -> bool:
    """
    Returns True only if the message is worth logging as an HR query.
    Filters out noise: single chars, emoji-only, bare email addresses.
    URLs are intentionally NOT filtered ‚Äî staff send links to orders,
    invoices and tickets, and these are valid queries worth logging.
    """
    if not text or len(text.strip()) <= 3:
        return False
        return False
    if re.match(r'^[\w\.-]+@[\w\.-]+\.\w+$', text.strip()):
        return False
    # Emoji-only: if stripping all ASCII leaves nothing meaningful
    ascii_only = re.sub(r'[^\x00-\x7F]+', '', text).strip()
    if len(ascii_only) == 0 and len(text.strip()) <= 5:
        return False
    return True
```

Then wrap the `/api/hr/log-query` call:

```python

# always calls log-query unconditionally

# AFTER:
if _should_log_hr_query(text):
    # existing log-query POST call here
    pass
# else: silently skip ‚Äî do not log noise
```

Do NOT remove or restructure the existing log-query POST call.
Only wrap it with the `if _should_log_hr_query(text):` guard.
