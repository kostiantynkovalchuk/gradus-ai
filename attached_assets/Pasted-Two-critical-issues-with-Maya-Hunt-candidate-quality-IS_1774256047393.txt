ISSUE 1: Telegram scraper returns job VACANCIES instead of candidate CVs

2. Improve the filter. A MESSAGE IS A VACANCY (not a CV) if it contains patterns like:
   - "шукаємо", "запрошуємо", "потрібен", "потрібна", "потрібні"
   - "вакансія", "відкрита позиція", "ми пропонуємо", "ми шукаємо"

   - "Надіслати резюме", "Писати в дірект", "Звертайтесь"
   
   A MESSAGE IS A CV (candidate looking for work) if it contains:
   - "шукаю роботу", "шукаю роботу", "розглядаю пропозиції"
   - "маю досвід", "мій досвід", "працював", "працювала"

   - Age + city pattern: "25 років, Дніпро"
   
   Logic:

   - If vacancy_signals > cv_signals → SKIP (it's a vacancy, not a CV)
