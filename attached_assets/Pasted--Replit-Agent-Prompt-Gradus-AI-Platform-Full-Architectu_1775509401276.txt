## PRODUCT 2 — MAYA HUNT (Internal AI Recruitment for AVTD)

### What It Is
Maya Hunt is an autonomous AI recruitment pipeline embedded in AVTD's HR infrastructure. It activates when HR posts a vacancy in a private Telegram supergroup and returns a scored shortlist in under 3 minutes.

### Pipeline Flow
```
HR posts vacancy in Telegram supergroup
    ↓
[1] PARSE — Claude Haiku extracts: position, city, salary, requirements → JSON
    ↓
[2] ACTION MENU — HR chooses: Search / Post / Both / Skip (inline keyboard)
    ↓
[3] SEARCH (parallel):
    ├── Work.ua public scraping (BeautifulSoup, no auth, up to 14 CVs)
    ├── 7 Telegram channels scan (Telethon read-only, 30 cap, CV vs vacancy filter)
    └── Robota.ua salary GraphQL (background task, cached)
    ↓
[4] SCORE — Claude Haiku scores each candidate 0–100 against vacancy requirements
    - City mismatch → hard cap 20/100
    - Minimum threshold: 35/100 (below = not shown)
    - safe_parse_json() repair for truncated responses
    ↓
[5] DELIVER — Top 5 cards in Telegram thread with buttons: ✅ ❌ 💾 🎯 Найняти
```

### Candidate Sources
| Source | Method | Volume |
|--------|--------|--------|
| Work.ua | Public scraping, Cloudflare bypassed via unauthenticated search | 14 CVs/run |
| Telegram channels | Telethon scan, channel_type=scan, is_cv_post() filter | 7 channels, 30 cap |
| Robota.ua | GraphQL authenticated (JWT token, expires April 12 2026) | Salary data only |

### Database (Neon PostgreSQL)
All Maya Hunt tables use sequential numbered migrations:
| Table | Migration | Purpose |
|-------|-----------|---------|
| hunt_vacancies | 011 | Parsed vacancy data |
| hunt_candidates | 011 | Scored candidates, HR decisions, hire tracking |
| hunt_sources | 011+016 | channel_type: scan/post/both |
| hunt_postings | 013 | Vacancy post attempt log |
| hunt_salary_data | 013+014 | Robota.ua employer vs candidate medians |
| schema_migrations | system | Migration tracking |

### Key Service Files (backend/services/)
| File | Responsibility |
|------|---------------|
| hunt_service.py | Main orchestration |
| hunt_vacancy_parser.py | Claude Haiku → vacancy JSON |
| hunt_tg_scraper.py | Telethon channel scanner |

## PRODUCT 3 — ALEX PHOTO REPORT (Field Merchandising Bot)

### What It Is
A Telegram bot used by AVTD field sales reps to photo-audit shelf placements at retail points. The rep sends shelf photos; the bot scores the display 0–100 using Claude vision.

### Flow
```
Rep sends /start → bot asks for store name
Rep sends photos (up to 5) → clicks "Готово"
Bot: "Аналізую 1 фото... Це займе ~15 секунд"
Bot returns scored report card:
  - Score: XX/100 ПРОЙДЕНО or НЕ ПРОЙДЕНО
  - Помилки (violations with rule IDs)
  - Доля полки горілки: XX% (норма ≥25%)
  - Інфо (informational, не впливає на оцінку): вино/коньяк % vs Phase 2 targets
  - Claude commentary on what was detected
  - ID звіту: #N
```

### Scoring Rules (examples observed in production)
- `1_108106` — Відсутній шелфстрипер GREENDAY при наявності конкурентських POS матеріалів
- `1_108608` — Частка полиці горілки менше 25% — провал звіту
- `NP59_200009734` — Відсутній загальний огляд всіх полиць (лівий+центр+правий край)

### Known Issues (from live testing, 19.03.2026)
- Bot sometimes classifies shelftockers on competitor bottles as "missing" when they shouldn't count
- AVTD wine and cognac brands (Adjari, Villa UA) not being identified correctly — counted as 0% or as competitor
- General overview photo requirement (NP59) fails when refrigerators form the visible boundary instead of the full shelf wall
- Cognac share calculation is mixing top shelf (spirits) with vodka shelf in some shots

### Status
Live and being tested by sales rep Александр. Several prompt/scoring logic fixes needed.
