# Replit Agent: Fix DATABASE_URL Split-Brain Across All Files

## Background

We discovered a critical split-brain bug: Replit's auto-provisioned PostgreSQL (`DATABASE_URL` pointing to Replit's internal `helium` host) contains different data than our production Neon database. This caused article IDs to diverge — the same article has different IDs in each database. When the Telegram approval system processed callbacks, it looked up the wrong article.

**Root cause:** Multiple files use `os.getenv("DATABASE_URL")` to connect directly via psycopg2, bypassing SQLAlchemy. If `DATABASE_URL` ever points to the wrong database (Replit's internal DB vs Neon production), these features break silently.

**Fix principle:** All database connections must prefer `NEON_DATABASE_URL` (explicit production DB) and fall back to `DATABASE_URL` only if `NEON_DATABASE_URL` is not set. This way:
- On **Render** (production): `NEON_DATABASE_URL` is not set → uses `DATABASE_URL` (which points to Neon) ✅
- On **Replit** (development): `NEON_DATABASE_URL` IS set → uses it directly, ignoring Replit's auto-provisioned DB ✅
