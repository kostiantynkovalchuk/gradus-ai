1. Migration 021 — model_deprecation_state
==================================================
Single-row table to persist the last alert state for content-based 
deduplication. NOT a log table (we don't need history; we need 
"what did we last alert about").

CREATE TABLE IF NOT EXISTS model_deprecation_state (
    id INTEGER PRIMARY KEY DEFAULT 1,
    last_check_at TIMESTAMPTZ,
    last_status TEXT,                    -- 'ok' | 'flagged' | 'fetch_failed'
    last_flagged_models JSONB,           -- ["claude-sonnet-4-6", ...] or []
    last_alert_sent_at TIMESTAMPTZ,      -- only set when alert actually fired
    CONSTRAINT single_row CHECK (id = 1)
);

INSERT INTO model_deprecation_state (id) VALUES (1) 
ON CONFLICT (id) DO NOTHING;

2. backend/services/model_deprecation_check.py
==================================================

Behavior contract:
- Fetch https://docs.claude.com/en/docs/about-claude/model-deprecations
- Scan for ai_models.SONNET, HAIKU, OPUS in the page
- For each match, check ±250 chars for retirement keywords:
  ["deprecat", "retire", "sunset", "end of life", "end-of-life"]
- Compare flagged set to last_flagged_models in DB
- Alert ONLY when the flagged set CHANGES, specifically:
  a) ok → flagged (new alert): "⚠️ Models newly flagged: [...]"
  b) flagged → flagged with NEW additions: "⚠️ Additional models flagged: [...]"
  c) flagged → ok (recovery): "✅ All previously-flagged models are clear"
  d) flagged → flagged with same set: silent, just update last_check_at
- On fetch failure: log only, do NOT alert, do NOT update last_status 
  (preserves the previous known state). If fetch fails 3+ months running, 
  that's caught by the OpenAI/Anthropic ping in api_token_monitor anyway.
- Always update last_check_at, even on fetch failure
- Update last_alert_sent_at only when an alert actually fires

Telegram message format (use existing notification_service style — 
plain text, no Markdown unless audit confirms it's supported):

⚠️ Claude model deprecation check

Active models flagged on Anthropic's deprecation page:
- SONNET = claude-sonnet-4-6
- OPUS = claude-opus-4-6

Review: https://docs.claude.com/en/docs/about-claude/model-deprecations
Update backend/services/ai_models.py to migrate.
