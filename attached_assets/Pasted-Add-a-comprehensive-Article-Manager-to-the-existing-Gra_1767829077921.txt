## 1. BACKEND API ENDPOINTS (FastAPI)

Add to `backend/main.py` or create `backend/routes/articles.py`:

### Endpoints needed:
```python
GET /api/admin/articles
  - Query params: limit, offset, search, status, date_from, date_to
  - Returns: list of articles with metadata
  - Include: article details + platform counts (facebook_posts, telegram_posts)

GET /api/admin/articles/{article_id}
  - Returns: full article details + all related platform posts
  - Include: facebook post IDs, telegram message IDs

DELETE /api/admin/articles/{article_id}
  - Deletes article + all related posts from all platforms
  - Returns: deletion summary (what was deleted)

POST /api/admin/articles/bulk-delete
  - Body: {"article_ids": [1, 2, 3]}
  - Deletes multiple articles
  - Returns: success/failure for each

POST /api/admin/articles/delete-by-date
  - Body: {"date_from": "2025-01-01", "date_to": "2025-01-07"}
  - Deletes articles in date range
  - Returns: count deleted

GET /api/admin/articles/export
  - Query params: same as GET articles
  - Returns: CSV/JSON export
```

### Features:
- Proper error handling with detailed messages
- Transaction support (rollback on failure)
- Deletion logging to database table or file
- Authentication middleware (check if admin user)
- CORS configuration for frontend
